# Lasala Fitness Studio

Membership and class booking app for Lasala Fitness Studio (Oradea, Romania).

## Stack

- **Next.js 15** (App Router, React Server Components)
- **TypeScript**, strict mode
- **Tailwind CSS** + shadcn/ui
- **Supabase** — Postgres, Auth, Storage, RLS
- **Resend** — transactional email
- **Vercel** — hosting
- **next-intl** — i18n (Romanian only at launch, EN-ready)

## Domain rules (single source of truth)

These are encoded as functions in `lib/booking/` and as constraints in `supabase/migrations/`:

- **Booking window:** sessions for week N+1 unlock at midnight on Sunday of week N
- **One per day:** a member can hold at most one active booking per calendar date
- **Plan required:** member must have an active, non-expired plan with `sessions_used < plan_total`
- **Cancellation window:** at least 3 hours before session start
- **Reschedule limit:** 2 reschedules per ISO week per member
- **Freeze:** 3–14 days per request, 14 days total per rolling 6 months, 48-hour advance notice, plan end date is extended by the freeze duration

## Payments

No payment provider integration. Members request a plan; an admin approves it after receiving payment via:

- **Bank transfer** (IBAN displayed on plan request page)
- **POS at the studio**
- **Cash at the studio**

A plan request stays in `pending` status until admin marks it `approved` (which activates the plan) or `rejected`.

## Auth

Supabase Auth with email/password and Google OAuth. Roles stored in a `profiles` table with `role IN ('member', 'admin')`. Admin access is gated by middleware reading the role from a server-side Supabase client; never trust the role from the client.

## Storage

Supabase Storage buckets:

- `progress-photos` — private, RLS so users only see their own
- `public-assets` — logo, hero images

Never store image data in the database.

## Setup (local)

1. `pnpm install`
2. Copy `.env.example` to `.env.local`, fill in Supabase + Resend keys
3. In a fresh Supabase project, run `supabase/migrations/*.sql` then `supabase/seed.sql` from the SQL editor
4. `pnpm dev`

## Deploy

- Vercel project linked to this repo, env vars set, custom domain attached
- Supabase project on the appropriate region (Frankfurt is closest to RO)
- Resend domain verified for `lasalastudio.ro` so emails send from `bookings@lasalastudio.ro` etc.

## What this replaces

A previous version of this app was generated by an AI tool and ran on Expo + FastAPI + MongoDB. We are not migrating data — the studio has only a handful of accounts, easier to recreate. Reusable assets (Romanian strings, GDPR document, email copy, dietary questionnaire fields, business constants) are in `assets/` for reference during the build.
